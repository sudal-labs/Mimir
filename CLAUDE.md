# CLAUDE.md

이 프로젝트의 규칙은 아래 4개 문서로 분리되어 관리됩니다.

- @OVERVIEW.md
- @STACK.md
- @CONVENTION.md
- @WORKFLOW.md

## 템플릿 사용법
- **새 프로젝트를 시작할 때는 OVERVIEW.md와 STACK.md의 "프로젝트 스택" 영역만 채우면 됩니다.**
- STACK.md의 "확장 스택"은 미리 체크하는 목록이 아니라, OVERVIEW.md의 요구사항/제한사항을
  분석해서 Claude가 직접 트레이드오프를 판단하고 채우는 영역입니다.
- 의미 있는 판단/제안은 WORKFLOW.md "의사결정 사이클"(제안🟡 → 검수 → 피드백 → 확정🟢)을 따릅니다.
  기각된 안건은 지우지 않고 🔴로 남기고, 이미 확정된 결정을 바꿀 때는 ⚪로 표시하고 새 항목을 추가합니다.
- 이 상태 표기는 STACK.md "확장 스택" 표뿐 아니라 OVERVIEW.md의 요구사항/제한사항 항목에도
  동일하게 적용됩니다. 새로운 요구사항/범위 확장을 제안할 때는 🟡로 추가하고, 사용자가
  확정하기 전까지는 진행 중인 제안으로 취급할 것.
- CONVENTION.md / WORKFLOW.md, 그리고 STACK.md 안의 "스택 판단 원칙"은 공통이라 수정 불필요합니다.
- 세션 시작 시 자동 로드되지 않으니, 요구사항이 애매하면 추측하지 말고 그 경로의 파일을 직접 열어서 확인할 것.
